# Navbar Design Index

## 20 Different Navigation Patterns for Kenikool UI

This directory contains detailed visual design specifications for 20 fundamentally different navigation patterns. Each design file includes:
- ASCII art visual diagrams
- Detailed layout specifications
- Dimensions and spacing
- Use cases
- Accessibility requirements
- Mobile behavior
- Animation specifications
- HTML structure examples

---

## Navigation Types (01-20)

### **01. Horizontal Top Bar** ✅
Classic website navbar — Logo left, links center/right, actions far right.  
**Best for**: Traditional websites, corporate sites, blogs

### **02. Sidebar Vertical** ✅
Full-height side navigation with collapsible icon-only mode.  
**Best for**: Dashboards, admin panels, web applications

### **03. Floating Centered** ✅
Detached rounded pill that floats above content.  
**Best for**: Portfolios, creative sites, minimal designs

### **04. Split Logo Center** ✅
Logo as central divider with links split left/right.  
**Best for**: Luxury brands, fashion, restaurants

### **05. Bottom Tab Bar** ✅
iOS/Android-style bottom navigation with icon tabs.  
**Best for**: Mobile apps, PWAs, touch-first interfaces

### **06. Mega Header** ✅
Two-row system with utility bar + main nav, supports mega menus.  
**Best for**: E-commerce, large content portals, enterprise sites

### **07. Breadcrumb Trail** ✅
Hierarchical path navigation (Home › Category › Page).  
**Best for**: Documentation, file systems, deep content structures

### **08. Drawer Hamburger** ✅
Hidden drawer triggered by hamburger menu icon.  
**Best for**: Mobile-first sites, minimal designs, space-constrained layouts

### **09. Command Bar** ✅
Keyboard-first search/navigation palette (⌘K style).  
**Best for**: Documentation, dashboards, power-user tools

### **10. Dot Navigation** ✅
Vertical scroll-spy dots for section jumping.  
**Best for**: Single-page sites, landing pages, storytelling layouts

### **11. Tab Bar Switcher** ✅
Segmented control with sliding active indicator.  
**Best for**: Settings panels, multi-view dashboards, profile sections

### **12. Accordion Tree** ✅
Collapsible nested navigation with expand/collapse sections.  
**Best for**: File explorers, complex site maps, documentation hierarchies

### **13. Radial/Circular Menu** ✅
Items arranged in circle around center trigger button.  
**Best for**: Mobile apps, creative tools, context menus

### **14. Grid/Tile Navigation** ✅
Card-based navigation grid with icons + descriptions.  
**Best for**: App launchers, service menus, portal homepages

### **15. Sticky Section Headers** ✅
Multiple sticky headers that stack as you scroll.  
**Best for**: Long-form content, reports, multi-chapter documents

### **16. Pagination Stepper** ✅
Step-by-step progress navigation with connected indicators.  
**Best for**: Multi-step forms, onboarding, checkout flows

### **17. Context Menu** ✅
Right-click/long-press contextual action menu.  
**Best for**: Editing interfaces, file managers, canvas tools

### **18. Speed Dial FAB** ✅
Floating action button with expandable sub-actions.  
**Best for**: Mobile apps, quick actions, creative tools

### **19. Carousel/Slider Navigation** ✅
Horizontal swipeable scroll with snap points.  
**Best for**: Image galleries, product showcases, card collections

### **20. Offcanvas Panel** ✅
Overlay panel that slides from viewport edge.  
**Best for**: Filters, shopping carts, detailed settings

## Design File Format

Each design file follows this structure:

1. **Visual Design** — ASCII art wireframe
2. **Detailed Layout** — Component breakdown with measurements
3. **Specifications** — Dimensions, colors, typography
4. **Use Cases** — When to use this pattern
5. **Key Features** — What makes it unique
6. **Mobile Behavior** — Responsive adaptations
7. **Accessibility** — WCAG compliance requirements
8. **Example HTML** — Implementation structure
9. **Animation** — Transition specifications
10. **Best Practices** — UX guidelines

## Component Naming Convention

- **Vanilla**: `<k-navbar-horizontal>`, `<k-navbar-sidebar>`, etc.
- **React**: `<NavbarHorizontal>`, `<NavbarSidebar>`, etc.

Each type is a separate component (not variants of one component).
